김태성 선거앱 PRD|1. 프로젝트 개요 & 목표
1. 프로젝트 개요

신안군수 예비후보 김태성의 공식 선거 앱으로, 유권자와 후보 사이의 직접 소통 채널을 구축합니다. 섬으로 이루어진 신안군의 지리적 특성과 고령 인구를 고려해 접근성과 직관성을 최우선으로 설계합니다. 단순한 홍보 앱을 넘어, 후보의 핵심 정책 비전을 유권자가 직접 체험·평가하고 의견을 전달할 수 있는 쌍방향 플랫폼을 지향합니다.

앱의 핵심 방향
  • 정책 중심 설계: 김태성 후보의 6대 공약이 앱의 가장 중요한 콘텐츠
  • 접근성 우선: 고령 유권자를 포함한 전 연령대 — 최소 글자 16sp, 버튼 48dp
  • 쌍방향 소통: 일방적 홍보가 아닌 군민 의견 수렴 및 전달
  • 신뢰성 확보: 팩트체크 기능으로 투명한 정치 문화 조성
  • 오프라인 대응: 도서 지역 특성 고려 — 네트워크 없이도 핵심 콘텐츠 열람
2. 핵심 목표 (Goals)
G-01 후보 정책 인지도 향상 — 신안군 전체 유권자의 60% 이상이 주요 공약을 인지
G-02 유권자 참여 확대 — 출마 선언 1개월 내 앱 다운로드 5,000건 달성
G-03 직접 소통 채널 구축 — 민원·의견 제출 후 48시간 내 응답 시스템 운영
G-04 신뢰도 제고 — 팩트체크 기능으로 허위 정보 대응 및 투명성 확보
G-05 선거일 투표율 제고 — Push 알림 및 D-Day 카운트다운으로 투표 참여 독려

3. 김태성 후보 핵심 정책 — 앱의 중심 콘텐츠

이 섹션은 앱 내 '정책 소개' 화면의 콘텐츠 기반입니다. 6개 정책 영역이 앱의 메인 콘텐츠를 구성하며, 각 정책은 카드 UI로 시각화됩니다.

🏝 관광 비전 — 섬 전체가 하나의 작품이 됩니다
  • 섬별 특화 콘텐츠 개발: 자연·문화·생활 기반의 차별화된 관광 콘텐츠 구축
  • 생활이 관광이 되는 구조: 주민의 삶과 생산 활동을 관광 자원으로 확장
  • 신안 전체 관광 네트워크: 섬 지역 접근성 확보 및 통합 관광 경험 제공
💰 경제 정책 — 군민의 지갑이 두꺼워지는 신안
  • 농수산물 제값 받는 유통구조 개선으로 어민·농민 소득 직접 증가
  • 관광·에너지 사업 주민 참여 확대 (수익 공유 구조)
  • 군민이 직접 체감하는 소득 증가 체계 마련
🏛 공직자 3대 공약 — 공정하고 존중받는 행정
  • 공무원 권한 회복: 측근 개입 차단, 정책·예산·행정 공직자 중심 민원 처리
  • 공직자 존중 행정: 직원 복지 우선 강화, 폭언·인격 모독 없는 조직 문화
  • 공정 인사 원칙: 줄서기·소문 인사 근절, 성과 중심 공정 인사, 다면평가 도입
🤝 열린 군정 — 군민이 주인인 신안
  • 정책 계획부터 군민 참여: 아이디어 공모·온라인 의견 수렴
  • 행정 실행과 평가에 군민 참여: 중간 점검 공개 보고
  • 열린 군정 실현: 정보 투명 공개, 소통 창구 상시 운영
📞 주민불편 ZERO 프로젝트 — 불편은 줄이고, 응답은 빠르게
  • 연중무휴 24시간 민원 응답: 전화·문자·카카오톡·앱 통합 접수
  • 생활 밀착형 지원: 어르신 병원 예약·이동 지원, 전기·수도·보일러 수리 지원
  • 즉각적인 현장 대응: 긴급 민원 즉시 출동, 읍·면 단위 전담 인력 배치
  • 통합 민원 플랫폼: 접수부터 처리까지 원스톱, 지연 시 자동 알림
❤️ 해피100 복지 — 모든 군민이 편안한 신안
  • 주거·돌봄 통합: 농촌형 주거단지, 실버타운·돌봄센터 운영
  • 의료·이동 지원: 교통약자 동행 서비스, 의료 동행 확대
  • 생활 돌봄 강화: 영양지원·공동식사, 방과 후 학교·돌봄 지원
  • 교육·미래 인재: 해외연수·교육 프로그램
  • 안전·무장애 환경: 경로당 안전시설, 배리어프리 생활환경 확대

4. 앱 6대 핵심 기능 요구사항

Flutter 기반 iOS·Android 동시 지원. 우선순위 P0 → P1 → P2 순서로 개발합니다.

기능명 주요 요구사항
①
정치인 프로필
랜딩 페이지 · P0
 이름·사진·선거구·소속 정당 표시 / 경력 타임라인 (학력·직책·수상) / 핵심 정책 입장 6개 카테고리 카드 / SNS 링크 (Facebook·Instagram·YouTube·카카오톡) / 앱 실행 시 첫 화면 / 오프라인 캐시 지원 (Hive)
②
정책 소개 ★
핵심 기능 · P0
 6개 정책 카드 목록 화면 / 정책 상세: 배경·목표·추진계획·기대효과 / 정책별 로드맵 타임라인 / 관련 사진·영상 갤러리 / 카카오톡·문자 공유 기능 / 유권자 별점 평가 (익명·Firestore 집계)
③
뉴스 피드
재방문 유도 · P1
 후보 관련 뉴스 RSS 자동 수집 / 캠프 공식 보도자료·영상 통합 / 카테고리 필터 (지역·정책·인터뷰) / 출처 언론사 표시 / 카카오톡·링크 공유 / shimmer 스켈레톤 로딩 UX
④
직접 연락
유권자 소통 · P1
 전화·이메일·카카오톡 원터치 연결 (url_launcher) / 선거 캠프 위치 지도 (Google Maps Flutter) / 문의 유형 선택 (정책·민원·자원봉사·취재) / UGC 필터링 (욕설·혐오 표현 검사) / 접수 확인 알림
⑤
설문 및 투표
참여형 · P2
 정책별 찬반·우선순위 설문 / 자유 의견 입력 (500자) / 실시간 집계 차트 (StreamBuilder + Firestore) / 익명 참여 보장 (sha256 해시 ID) / 지역(읍·면)별 의견 분류 / 중복 참여 방지
⑥
팩트체크 & 알림
신뢰도 · P2
 발언·공약 팩트체크 (GREEN·YELLOW·RED 등급) / 출처 명시 필수 / 유세 일정·주요 이벤트 Push 알림 (FCM) / 선거일 D-Day 카운트다운 위젯 / 투표소 위치 안내 (행정안전부 API) / 알림 항목별 개인화 ON/OFF

6. 앱스토어 정책 준수 핵심 원칙

❌ 절대 구현 금지 (Google Play / App Store 위반)

  • 앱 내 정치 기부금·후원금 직접 결제 버튼 — 외부 링크(url_launcher)로만 안내
  • 상대 후보 실명 비방·공격 콘텐츠 삽입
  • 설문 참여 시 이름·연락처 등 개인 식별 정보 수집
  • 위치 데이터 서버 전송 (투표소 안내 시 기기 내 처리만 허용)
  • Google AdMob 등 광고 SDK 삽입 (정치 앱 광고 정책 위반)

✅ 반드시 구현

  • 익명 설문 — sha256(deviceId + surveyId) 해시 ID만 사용
  • UGC(사용자 입력) 서버 측 금칙어 필터링 적용
  • WCAG 2.1 AA 접근성 — 최소 터치 영역 48×48dp
  • Firebase Analytics 익명 집계만 허용, 개인 식별 이벤트 금지
  • 연령 등급 4+ (Apple) / 전체이용가 (Google Play)
